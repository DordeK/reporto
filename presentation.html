What We Built

One platform. Zero SQL.

Reporto connects to your Peppol inbox, detects anomalies automatically, and lets anyone on your team query financial data in plain English.

  • 📥 Ingest from anywhere — Upload UBL XML, pull from Peppol inbox, or receive via email. Deduplication by SHA-256 hash.
  • 🚨 10 automatic anomaly checks — VAT mismatches, duplicate invoice numbers, missing VAT IDs, reverse-charge violations, future-dated invoices, and more. Runs on every import.
  • 💬 Natural language reports — Type "Show total VAT per supplier this quarter" and get a chart, SQL, and AI narrative. No BI tool, no analyst required.
  • 📋 Belgian VAT return (Intervat) — Auto-calculates grid values and generates the submission XML. One click.
  • 📤 Send outgoing invoices via Peppol — Full two-step workflow: build UBL, validate, deliver to any Peppol-registered receiver.

Security & Validation

We trust nothing the LLM says

The AI generates a report definition — then a 5-layer pipeline validates it before a single query runs.

1
LLM → ReportDefinition JSON
GPT-4o-mini produces a structured JSON object — not raw SQL. Fields, filters, groupings, ordering.
2
Pydantic field allowlist
Every field name is checked against ALLOWED_FIELDS — a hardcoded whitelist. Any unknown field → rejected instantly.
3
Parameterised SQL only
SQLAlchemy text() with named params. Zero raw string interpolation. No injection surface.
4
EXPLAIN before execute
The compiled SQL is EXPLAIN-checked on PostgreSQL before it ever runs. Bad plans fail fast.
5
Completeness + reconciliation + quality score
Result is checked for dataset completeness, reconciled against known totals, and scored 0–100. Score and warnings are returned to the user.
Result: The LLM cannot produce a harmful query — even if the prompt tries to make it. The security boundary is in code, not in the prompt.
