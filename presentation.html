Report Validation

We trust nothing the LLM says

The AI produces a report definition — a 6-layer pipeline validates it before any query runs.

1
Tool calling dispatch
AI picks the right tool and dispatches it automatically.
2
Parameterised SQL only
SQLAlchemy named params. Zero raw string interpolation. No injection surface.
3
EXPLAIN before execute
PostgreSQL EXPLAIN check before query runs. Bad plans fail fast.
4
Dataset completeness check
Result verified against known invoice totals. Missing data flagged, not silently ignored.
5
Data quality score 0–100
Reconciliation + anomaly impact analysis. Score and warnings returned to the user.
The LLM cannot produce a harmful query — even if the prompt tries to make it. The security boundary lives in code, not in the system prompt.
